# 💳 Credit Card Fraud Detection

End-to-end fraud detection pipeline on 284,807 real-world transactions. Covers EDA, preprocessing, class imbalance handling, comparison of 6 models, and a live Streamlit dashboard.

---

## 📊 Results

| Model                        | Precision (Fraud) | Recall (Fraud)  | F1 (Fraud)      |
| ---------------------------- | ----------------- | --------------- | --------------- |
| Logistic Regression          | 0.871             | 0.602           | 0.712           |
| SGD + LogReg                 | 0.842             | 0.556           | 0.670           |
| SVM (RBF)                    | 0.937             | 0.645           | 0.764           |
| Random Forest                | 0.945             | 0.787           | 0.856           |
| **XGBoost (tuned) ✅** | **0.963**   | **0.800** | **0.872** |

> All models evaluated on a held-out test set (20%) with 5-fold cross-validation on the training set.

---

## 🔍 Problem

The dataset is severely imbalanced — only **0.17% of transactions are fraudulent** (492 out of 284,807). Standard accuracy is misleading; the focus is on F1 score  for the fraud class.

---

## 🛠 Pipeline

**1. EDA** (`small_eda_notebook.ipynb`)

- Distribution analysis of all 28 PCA features + Amount + Time
- Class imbalance visualization
- Outlier detection in the Amount column

**2. Preprocessing** (`data_scaling_notebook.ipynb`)

- `RobustScaler` on `Amount` and `Time` (resistant to outliers, unlike StandardScaler)
- Stratified 80/20 train-test split
- Scaled datasets saved as CSV for reproducibility

**3. Modeling** (`unbalance_modeling_notebook.ipynb`)

- 6 models benchmarked: Logistic Regression, SGD, Random Forest, SVM, XGBoost
- 5-fold cross-validation on training set
- SMOTE applied to handle class imbalance
- XGBoost tuned with GridSearchCV

**4. Streamlit App** (`app.py`)

- Overview: class distribution, amount distribution, SMOTE explanation
- Model performance: confusion matrix, ROC curve, feature importance
- Live prediction: simulate a transaction and get an instant fraud score

## 🚀 Run locally

````bash
git clone https://github.com/kaiokasa/fraud-detection
cd fraud-detection
pip install -r requirements.txt
streamlit run app.py
````

> You'll also need `creditcard_train_scaled.csv`, `creditcard_test_scaled.csv`, and `best_xgb_model.pkl` in the same folder. The CSVs are generated by `data_scaling_notebook.ipynb` from the [Kaggle dataset](https://www.kaggle.com/datasets/mlg-ulb/creditcardfraud).
